# brainstorm: 重新设计网页前端界面

## Goal

重新设计 CanvasToAPI 项目的网页前端界面，提升用户体验、视觉效果和可用性。

## What I already know

**当前技术栈**：

- Vue 3 + Vue Router + Element Plus
- Vite 构建工具
- Less CSS 预处理器
- i18n 国际化支持（中英文）
- 主题切换功能

**当前页面结构**：

- LoginPage：登录页面
- StatusPage：主状态页面，包含 3 个 tab（home/settings/logs）
  - **home tab**：服务状态、会话池信息、代理设置状态（流式模式、强制思考、强制网页搜索、强制URL上下文等）
  - **settings tab**：设置面板（具体功能待确认）
  - **logs tab**：实时日志查看
- NotFound：404 页面

**当前布局特点**：

- 垂直侧边栏导航（图标菜单）
- 主要内容区
- 语言切换按钮
- 登出按钮
- 大量 SVG 图标内嵌

**项目背景**：

- 这是一个 Canvas to API 代理服务器项目
- UI 用于监控系统状态、查看日志、管理设置
- 需要认证才能访问主功能

## Assumptions (temporary)

- 用户希望保留现有的核心功能（状态监控、日志、设置）
- 现有的技术栈（Vue 3 + Element Plus + Less）将继续使用
- 需要支持移动端响应式设计
- 需要保留国际化功能

## Design Decisions

**设计风格**：技术感暗色主题（Real-Time Monitoring）

- 参考：DevOps 监控工具、实时数据平台
- 特点：暗色模式优先、状态指示器动态效果、监控语义色
- 原因：适合技术监控工具，视觉专业且现代

**配色方案**：

- 主色调：深黑 `#18181B`
- 次要色：中灰 `#3F3F46`
- CTA色：品红 `#EC4899`（创意对比）
- 背景：亮色 `#FAFAFA` / 暗色 `#09090B`
- 文字：深色 `#09090B` / 亮色 `#FAFAFA`
- 状态色：成功 `#22C55E`、警告 `#F97316`、错误 `#EF4444`

**排版方案**：

- 标题字体：Bodoni Moda（高对比度优雅）
- 正文字体：Jost（几何现代）

**视觉特点**：

- 状态指示器动态效果（脉冲动画）
- 实时数据流视觉提示
- 监控特有的颜色语义（红/橙/绿）
- 暗色模式优先设计
- 线性图标（Lucide Icons / Heroicons outline）

**图标风格**：

- 线性图标，stroke-width: 1.5-2px
- viewBox: 24x24，尺寸: w-6 h-6
- 禁止使用 emoji 作为图标

**布局结构**：保持侧边栏导航（优化设计）

- 收窄宽度、增加留白
- 使用更轻量的图标和文字
- 更简洁的视觉呈现

## Open Questions

**Preference Questions**:

1. 是否需要改进现有功能或交互流程？
2. 移动端适配优先级如何？

## Requirements (evolving)

**核心功能保留**：

- 状态监控页面（服务状态、会话池、代理设置）
- 设置管理面板
- 实时日志查看
- 国际化支持（中英文）
- 暗色模式支持

**设计优化方向**：

- 极简简约风格（Notion/Linear 风格）
- 保持侧边栏导航，优化视觉设计
- 大量留白、清晰的信息层级
- 简洁的线条和轻量化的图标
- 改进整体视觉效果和品牌形象

**优化范围**：

- ✅ 视觉重设计（配色、间距、排版、图标风格）
- ✅ 信息架构优化（重新组织信息层级，简化状态展示）
- ✅ 交互体验改进（动画过渡、悬停效果、状态反馈）
- ✅ 全面考虑用户体验的各个方面

**MVP 范围**：

- 重设计现有核心功能
- 确保移动端响应式可用
- 为未来扩展预留空间（数据可视化、离线体验等）

**移动端适配策略**：

- 桌面优先设计
- 移动端侧边栏转换为抽屉式菜单（drawer）
- 通过汉堡按钮触发导航菜单
- 保持核心功能可访问
- 基础响应式支持（375px, 768px, 1024px, 1440px）

## Technical Approach

**设计系统实现**：

1. 更新 `variables.less` — 定义新的 CSS 变量系统
2. 创建统一的图标组件 — 封装 Lucide/Heroicons
3. 实现暗色主题切换逻辑
4. 优化信息架构 — 简化状态展示、改进卡片布局
5. 添加过渡动画 — hover、tab 切换、抽屉菜单

**响应式策略**：

- 桌面端：侧边栏固定显示
- 平板端（<1024px）：侧边栏可折叠
- 移动端（<768px）：侧边栏转为抽屉菜单 + 汉堡按钮

**组件重构**：

- 抽取可复用的状态卡片组件
- 统一图标使用方式
- 优化 SVG 内联方式（考虑使用图标库）
- 改进状态指示器（添加脉冲动画）

**性能优化**：

- 使用 CSS 变量减少样式重复
- 优化 SVG 图标加载
- 确保 transition 性能（使用 transform/opacity）

## Technical Notes

**相关文件**：

- `ui/app/App.vue` - 根组件
- `ui/app/pages/StatusPage.vue` - 主状态页面（较大，25969 tokens）
- `ui/app/pages/LoginPage.vue` - 登录页面
- `ui/app/router/index.js` - 路由配置
- `ui/app/styles/global.less` - 全局样式
- `ui/app/styles/variables.less` - 样式变量
- `ui/app/utils/i18n.js` - 国际化工具
- `ui/app/utils/useTheme.js` - 主题工具

**技术约束**：

- 必须兼容 Vue 3
- 必须使用 Element Plus 组件库
- 必须支持 Vue Router
- 样式必须使用 Less 预处理器
- 需要考虑 Vite 构建配置

**Element Plus 版本**：^2.7.2

**图标库推荐**：

- Lucide Icons（推荐）：轻量、现代、完整
- Heroicons（备选）：简洁、适合极简风格
- 实现方式：SVG 组件或图标字体

**性能要求**：

- 首屏加载时间无明显增加
- 动画帧率保持 60fps
- 移动端流畅度良好
