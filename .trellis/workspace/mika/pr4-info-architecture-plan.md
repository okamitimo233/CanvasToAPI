# PR4: 信息架构优化计划

## 目标

优化 StatusPage 信息架构,提升用户体验和可读性。

## 当前问题

### 1. 信息重复

- 代理设置状态在 Home tab 和 Settings tab 重复显示
- 用户需要切换 tab 才能修改设置

### 2. 信息层级不清晰

- Home tab 混合了状态展示和配置信息
- 浏览器会话列表与上面卡片视觉层级一致

### 3. Tab 切换体验欠佳

- 无平滑过渡动画
- Logs tab 滚动位置恢复逻辑复杂

### 4. 状态展示可简化

- 服务状态卡片内容过多
- 会话池卡片与浏览器会话列表信息重叠

## 优化方案

### Phase 1: 简化信息展示 (当前)

**目标**: 减少 Home tab 信息密度,提高可读性

**改动**:

1. ✅ 合并"服务状态"和"会话池"为一个"系统概览"卡片
2. ✅ 移除 Home tab 中的"代理设置状态"卡片(用户可在 Settings tab 查看/修改)
3. ✅ 优化浏览器会话列表标题层级

**影响范围**: 仅 UI 层面,无后端改动

**风险**: LOW - 只涉及模板和样式修改

### Phase 2: 改进 Tab 切换

**目标**: 提升交互流畅度

**改动**:

1. 添加 Tab 切换过渡动画(fade + slide)
2. 简化 Logs 滚动位置恢复逻辑

**影响范围**: 仅前端交互

**风险**: LOW - 可通过 CSS transition 实现

### Phase 3: 数据状态反馈

**目标**: 提升用户体验

**改动**:

1. 添加 loading 状态
2. 改善错误提示
3. 添加空状态设计

**影响范围**: 前端状态管理

**风险**: MEDIUM - 需要添加新的状态变量

## 实施原则

- **渐进式优化**: 每个 phase 独立提交,可独立回滚
- **保持功能完整**: 不删除任何现有功能
- **向后兼容**: 确保移动端适配不受影响
- **性能优先**: 动画使用 transform/opacity,避免触发重排
